1 Quickstart

Get your API key

Sign up for a free account and get your API key in seconds. No credit card required.

Make your first request

Look up any US property by address:

curl https://api.parceldata.ai/v1/properties/address \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "street": "123 Main St",
    "city": "Austin",
    "state": "TX"
  }'

Explore the response

3 Authentication

Authenticate requests with your API key via the Authorization header or X-API-Key header:

# Bearer token (recommended)
Authorization: Bearer pk_live_your_key_here
# Or X-API-Key header
X-API-Key: pk_live_your_key_here

API Reference

Base URL: https://api.parceldata.ai/v1

Endpoints

GET /v1/properties/{parcel_id}

Lookup a property by its parcel ID.

GET /v1/properties/address

Lookup a property by street address, city, and state.

GET /v1/properties/coordinates

Lookup a property by latitude and longitude.

POST /v1/properties/search

Search properties with filters (city, state, zoning, price range, etc.).

POST /v1/properties/batch

Batch lookup up to 100 properties in a single request.

GET /v1/analytics/comparables

Get comparable sales for a property.

GET /v1/analytics/market-trends

Get market statistics for a geographic area.

Response Format

All responses are JSON and include a data_quality object:

{
  "data": { /* property data */ },
  "data_quality": {
    "score": 0.92,          // 0-1 composite score
    "completeness": 0.95,    // fields populated
    "accuracy": 0.98,        // cross-source verified
    "timeliness": 0.85,      // freshness factor
    "freshness_hours": 4,    // hours since last update
    "sources": ["travis_cad", "actris_mls"],
    "confidence": "high"     // high | medium | low
  },
  "meta": {
    "request_id": "req_abc123",
    "tokens_used": 1250
  }
}

Detail Tiers

Control response size with the ?detail= query parameter:

Tier Tokens Includes
micro500-1KID, price, basic stats
standard2K-4KFull property details
extended8K-16KProperty + market context
full32K+Everything + documents

SDKs

Python SDK

from parceldata import ParcelData

# Initialize client
pd = ParcelData(api_key="pk_live_your_key")

# Lookup a property
prop = pd.lookup("123 Main St, Austin, TX")
print(prop.address)       # "123 Main St, Austin, TX 78701"
print(prop.zoning)        # "R1-SF"
print(prop.lot_sqft)      # 8500
print(prop.data_quality)  # 0.92

# Search properties
results = pd.search(
    city="Austin",
    state="TX",
    zoning="R1-SF",
    min_sqft=2000,
    max_price=500000,
    detail="standard"
)

for prop in results:
    print(f"{prop.address}: ${prop.tax_assessed:,}")

# Get comparables
comps = pd.get_comparables("TX-TRAVIS-12345", radius_miles=1)

# Batch lookup
addresses = ["123 Main St, Austin, TX", "456 Oak Ave, Dallas, TX"]
batch = pd.batch_lookup(addresses)

TypeScript SDK

import { ParcelData } from '@parceldata/sdk'

const pd = new ParcelData({ apiKey: 'pk_live_your_key' })

// Lookup a property
const property = await pd.lookup({
  address: '123 Main St, Austin, TX'
})

console.log(property.zoning)              // "R1-SF"
console.log(property.dataQuality.score)    // 0.92

// Search with filters
const results = await pd.search({
  city: 'Austin',
  state: 'TX',
  zoning: 'R1-SF',
  detail: 'micro'  // minimize token usage
})

MCP Server

Add ParcelData as an MCP server for any MCP-compatible agent:

// mcp_config.json
{
  "mcpServers": {
    "parceldata": {
      "command": "npx",
      "args": ["@parceldata/mcp"],
      "env": {
        "PARCELDATA_API_KEY": "pk_live_your_key"
      }
    }
  }
}

Examples

Property Search Agent

Build an agent that can find properties matching specific criteria:

from parceldata import ParcelData

pd = ParcelData(api_key="pk_live_...")

# Find undervalued properties near good schools
results = pd.search(
    city="Austin",
    min_school_rating=8,
    max_price=400000,
    min_lot_sqft=6000,
    detail="standard"
)

for p in results:
    comps = pd.get_comparables(p.parcel_id)
    avg_comp = sum(c.price for c in comps) / len(comps)
    if p.tax_assessed < avg_comp * 0.8:
        print(f"Potential deal: {p.address}")

Data Quality

Every ParcelData response includes a quality score so agents can make informed decisions about data reliability.

Component Description Range
scoreComposite quality score0-1
completenessPercentage of fields populated0-1
accuracyCross-source verification0-1
timelinessFreshness decay factor0-1
freshness_hoursHours since last data update0+
confidenceOverall confidence levelhigh/medium/low
